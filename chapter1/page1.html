HTML 文件结构(Document Structures)

一个HTML文件包含三个部分：

  1. 一个标明HTML 版本信息的行
  2. 一个放置背景信息声明的头部HEAD
  3. 页面内容的主体部分，由BODY元素或FRAMESET元素表示

一个合法的HTML文件首先声明所遵行的HTML版本，可以是以下三种 形式之一：

  • <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN"
        "http://www.w3.org/TR/html4/strict.dtd">

  • <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN"
        "http://www.w3.org/TR/html4/loose.dtd">

  • <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Frameset//EN"
        "http://www.w3.org/TR/html4/frameset.dtd">


后续HTML文件部分的结构如下： 

<HTML>
<HEAD>
     <title>, <base>, <link>,  <meta> , ...
</HEAD>
<BODY>
    HTML 文件的正文写在这里... ...
</BODY>
</HTML>

大约在2004年以后，随着WEB 2.0标准的渐渐被采用， 为了向将来兼容XML的一些标准，页面的版本信息申明部分最好采用如下内容：

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.1//EN" "http://www.w3.org/TR/xhtml11/DTD/xhtml11.dtd">

相应的在HTML标记里要加上名域空间的申明语句：

<html xmlns="http://www.w3.org/1999/xhtml" xml:lang="en">

TITLE元素

在HEAD部分，很多与页面有关的背景信息写在这里。而且每个 规范的HTML文件的HEAD部分必须包含title元素。title的内容最好能 比较清楚地代表页面内容。

META元素

meta是用来在HTML文档中模拟HTTP协议的响应头报文。 meta元素有这么几个属性：name, content,http-equiv, scheme。 name=″.....″ 表示META信息的名称，HTTP－EQUIV=″....″ 表示HTTP响应的 标题头，content=″....″ 表示META信息的具体内容，scheme=″...″ 表示 META信息的图解。HEAD部分可以包含多个meta元素，每一个meta元素指定一个属性名/属性值对。 例如：

<META name="Author" content="张三">

meta元素的应用之一就是指定页面的关键词以便于搜索引擎找到 该页面。下面的例子可以分辨不同语言下的关键词：


<-- For speakers of US English -->
<META name="keywords" lang="en-us"
         content="vacation, Greece, sunshine">
<-- For speakers of British English -->
<META name="keywords" lang="en"
         content="holiday, Greece, sunshine">
<-- For speakers of French -->
<META name="keywords" lang="fr"
         content="vacances, Greece, soleil">






有效提高网页被搜索到的概率的做法

<meta name="GENERATOR" content="Microsoft FrontPage 3.0">

这表示该网页由什么编辑器写成。

可以用meta元素指定文件所用的字符集：

<META http-equiv="Content-Type" content="text/html; charset=GB2312" >

它起的作用是定义网页所使用的语言,浏览器根据此项，就可以选择正确的语言编码，而不需要读者 自己在浏览器里选择。当访客浏览网页时,浏览器会自动识别并设置网页中的语言,如果网页设置的是GB码, 而客户端没有安装GB码,这时网页就会呈现客户端所设置的浏览器默认语言。同样的,如果该网页是英语,那么charset=en。 GB2312是指简体中文，而台湾BIG5内码的主页则是用BIG5。

<META HTTP-EQUIV="refresh" CONTENT="60; URL=new.htm">

　　浏览器将在60秒后，自动转到new.htm。你可以利用这个功能，制作一个封面，在若干时间后，自动带读者来到你的目录页。

　　如果URL项没有，浏览器就是刷新本页。这就实现了WWW聊天室定时刷新的特性。

<META HTTP-EQUIV="Pragma" CONTENT="no-cache">

强制性调用网上的最新版本。浏览器为了节约时间，在客户端 硬盘上保存一个以前访问过的网页文件的临时版本。在客户要重新访问该文件时，浏览器会直接显示客户机硬盘上的文件 ，而不是服务器上的。如果你想让读者每次都通过直接访问服务器端的页面来阅读，就加上这句话。

<META HTTP－EQUIV="expires" CONTENT="TUE,15.MAR 2005 00:00 GMT">

META可以设置网页到期的时间,也就是说,当客户在 浏览器中浏览网页时,首先查看本地缓冲里的页面,那么当客户浏览某一网页,而客户机的 缓冲里又有时,那么浏览器会自动浏览缓冲区里的页面,直到META中设置的时间到期, 这时候,浏览器才会去取得新页面。

<META HTTP－EQUIV="set-cookie" CONTENT="TUE,15.MAR 2005 00:00 GMT">

设定cookie的有效期限，如果网页过期，存盘的cookie将被删除。

<META HTTP－EQUIV="windows-Target" CONTENT="_top">

强制页面在当前窗口中以独立页面显示，可以防止自己的网页被别人当作一个frame页调用。

关于协议类型



LINK元素

link元素只能出现在HEAD部分，用来表示当前文件与其它文件的 连结关系，但不会有连结按钮。它的常用属性有href, rel, rev, charset。href指明要连接 的文件的URL，rel（relationship)表示本文档到所链接的资源的指向关系，rev(reverse)定义了 逆向的关联，表示来自链接的关系，charset表示链接对象所使用的字符集。

链接类型的属性值及其说明

Alternate      定义替换版本
Stylesheet      指向外部样式表文件
Start      指向一组文档集合的第一篇文档
Next      在一组线性文档中指向下一篇文档
Prev      在一组线性文档中指向前一篇文档
Index      指向提供当前文档目录的一篇文档
Glossary      指向提供当前文档词汇表的一篇文档
Copyright      指向提供当前文档版权声明的一篇文档
Chapter      指向作为一系列文档章节的一篇文档
Section      指向作为一系列文档章节的一篇文档
Appendix      指向作为一系列文档附录的一篇文档
Help      指向作为一系列文档帮助的一篇文档
Bookmark      指向一个书签

下面是一个应用link元素的例子，当前文件是"Chapter2.html", 其中"Index", "Next", 以及 "Prev" 是具体链接关系的类型。 


<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN"
   "http://www.w3.org/TR/html4/strict.dtd">
<HTML>
<HEAD>
  <TITLE>Chapter 2</TITLE>
  <LINK rel="Index" href="../index.html">
  <LINK rel="Next"  href="Chapter3.html">
  <LINK rel="Prev"  href="Chapter1.html">
</HEAD>
...the rest of the document...

BASE元素

BASE元素定义文档的基础URI，为文档中其他使用相对URI表示地址的方法提供寻址的基础。使用BASE元素时，必须是在没有任何其他元素指向任何外部文 件之前应用。由BASE元素指定的路径信息只影响出现BASE元素的 文件本身里的链接。

下面的例子表示了BASE元素和页面中的链接的关系： 



<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN"
   "http://www.w3.org/TR/html4/strict.dtd">
<HTML>
 <HEAD>
   <TITLE>Our Products</TITLE>
   <BASE href="http://www.aviary.com/products/intro.html">
 </HEAD>

 <BODY>
   <P>Have you seen our <A href="../cages/birds.gif">Bird Cages</A>?
 </BODY>
</HTML>

其中的相对路径URI "../cages/birds.gif" 相当于:


     http://www.aviary.com/cages/birds.gif

语言字符集(Charsets)的信息

<meta http-equiv="Content-Type" content="text/html;charset=#">

#=
us-ascii, iso-8859-1, x-mac-roman, iso-8859-2, x-mac-ce,
iso-2022-jp, x-sjis, x-euc-jp,
euc-kr, iso-2022-kr,
gb2312, gb_2312-80,
x-euc-tw, x-cns11643-1, x-cns11643-2, big5

可在 HTML 文件中设置 MIME(Multipurpose Internet Mail Extensions，即多功能Internet 邮件扩充服务） 字符集信息。
客户在浏览页面时，自己可以在浏览器的选项菜单内选择相应的语言编码(language encoding)。
但是如果 HTML 文件里写明了设置，浏览器就会自动设置语言选项。
尤其是主页里用到了字符实体(entities)，则该主页就应该写明字符集信息。
